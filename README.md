# Machine Learning Implementation Library 🤖
This repository is a comprehensive collection of Machine Learning algorithms, feature engineering techniques, and end-to-end pipelines implemented using Python and Scikit-Learn.

## 📁 Repository Overview
The repository is structured into several key areas of the Machine Learning lifecycle:

### 1. Supervised Learning - Regression & Classification
* **Linear Models**: Implementation of Simple, Multiple, Polynomial, Ridge, Lasso, and ElasticNet Regression.
* **Gradient Descent**: Custom implementations of Batch, Mini-Batch, and Stochastic Gradient Descent (SGD).
* **Classification**: Logistic Regression (including Perceptron trick), SVM, Naive Bayes, and Softmax Regression.
* **Trees**: Decision Tree Classifiers and Regressors.

### 2. Unsupervised Learning - Clustering
* **K-Means**: Basic and advanced K-Means clustering implementations.
* **Hierarchical**: Agglomerative and Hierarchical clustering using Scikit-Learn.
* **Density-Based**: DBSCAN implementation for non-linear data clusters.

### 3. Ensemble Techniques
* **Bagging**: Implementations of Bagging Classifiers and Regressors.
* **Boosting & Voting**: Voting Classifiers and Regressors for improved model performance.

### 4. Feature Engineering & Preprocessing
* **Handling Data**: Managing missing values (Random Sample Imputation), mixed variables, and date-time data.
* **Transformers**: Power Transforms (Box-Cox, Yeo-Johnson), Function Transformers, and Binarization.
* **Encoding**: One-Hot, Ordinal, and Label encoding for numerical and categorical data.
* **Scaling**: Standardization and Normalization techniques.

### 5. Automation & Pipelines
* **Scikit-Learn Pipelines**: Multi-stage pipelines for streamlined data processing and modeling.
* **Profiling**: Automated data profiling reports using Pandas Profiling.

## 🛠️ Tech Stack
* **Language**: Python
* **Environment**: Jupyter Notebooks
* **Libraries**: NumPy, Pandas, Scikit-Learn, Matplotlib, Seaborn, Pillow, Tkinter

## 🚀 How to Navigate
1. Browse the `ipynb` files to see step-by-step implementations.
2. Refer to the `.csv` files for the datasets used in the case studies and clustering examples.
3. Check the `pipelines` notebooks to see how to integrate preprocessing and modeling.
